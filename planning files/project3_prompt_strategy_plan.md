# LearnMateAI 垂直切片开发 Prompt 计划表

基于你的 Sprint Plan 以及我们讨论的“**端到端迭代（垂直切片交付）**”策略，我为你制定了这份开发和 Prompt 指南。

**核心原则：**
1. **先通后精**：先用最不讲究样式的组件打通前后端数据流，再加 Tailwind CSS 润色。
2. **职责分离**：一次只让 AI 做一件事（例如：先写路由，再写组件，最后再加样式），防止它修改无关代码导致冲突。
3. **严格遵循规则**：所有后端新功能强制遵循 TDD 开发模式，前端组件强制遵守 Tailwind 规范和无 `any` 类型限制。

以下是尚未完成的核心特性的开发与 Prompt 拆解：

---

## 阶段一：补全 Sprint 1 剩余特征 (模块展示与资料管理)

### 1.1 S3 资料上传与模块基础展示
目前后端的 Module CRUD 已完成，还缺 S3 逻辑以及所有的前端展示。我们从这里开始填补盲区。

#### **步骤 1：S3 后端上传 API (Backend)**
*   **任务**：在后端实现连接真实的 AWS S3（或 Mock S3 / Minio），允许教师上传 PDF/图片等教学资料至对应模块，并把 URL 存入数据库。
*   **AI 指令 (Prompt)**：
    > "请结合现有的 `server/src/routers/modules.py`，使用 TDD 的方式，帮我新增一个上传资料到 S3 的后端 API（`POST /modules/{module_id}/materials`）。要求：  
    > 1. 先在 `tests/` 目录下写好对应的红灯测试。
    > 2. 为 Module 数据结构（Schema）增加 'Learning Objectives（学习目标）' 字段，以及一个专门的 **'受众背景/情感敏感度 (Audience Context / Sensitivity)'** 字段（用于约束 AI 生成的内容，比如：班上有同学是孤儿,避免使用父母举例;班上很多少数族裔同学,可以多用少数族裔为正面例子等）。
    > 3. 只做后端的逻辑和更新，**不要碰前端代码**。 
    > 4. 妥善处理 Boto3/S3 的环境变量接入（遵循不硬编码凭证的安全规则）。"

#### **步骤 2：教师端模块管理 UI (骨架版) (Frontend)**
*   **任务**：写一个极简的前端页面，能调用 GET 接口渲染现有的模块列表，并包含一个简单的创建表单和文件上传输入框。
*   **AI 指令 (Prompt)**：
    > "现在我们来写前端。请你帮我创建一个功能性的 React 骨架组件 `InstructorModuleDashboard.tsx`，调用后端的 `/modules` 相关接口获取列表，并提供一个基础的表单用于创建模块和上传文件资料。要求：
    > 1. 无需关注美观，没有任何花里胡哨的 CSS。
    > 2. 只需要确保 `fetch` 或 `axios` 调用成功，网络请求的状态处理清楚。
    > 3. 必须是严格的 TypeScript，禁止使用 `any`。"

#### **步骤 3：学生端模块浏览与笔记上传 UI (Frontend)**
*   **任务**：前端展示已有模块供学生浏览，并对接在上文已经写好的 `POST /modules/{module_id}/notes` 的后端接口。
*   **AI 指令 (Prompt)**：
    > "为学生端编写 `StudentModuleView.tsx` 组件。要求：
    > 1. 列表展示可用的课程模块。
    > 2. 提供清晰的『查看/下载课程资料 (View/download materials)』入口。
    > 3. 为每个模块加上一个 '上传学习笔记' 的基础输入框和按钮，对接响应的 `/notes` 接口。
    > 3. 保持骨架级别的简单直接，重点在于确保数据流能顺畅通过验证存入数据库。"

#### **步骤 4：全局 UI 润色 (Frontend Polish)**
*   **任务**：为前面建立的骨架组件全部装配 Tailwind CSS 和现代化动效。
*   **AI 指令 (Prompt)**：
    > "我们已经走通了师生模块管理的前后端数据流。现在请全面美化 `InstructorModuleDashboard` 和 `StudentModuleView`。
    > 1. 严格使用 Tailwind CSS 工具类。
    > 2. 实现现代化、动态的高级感设计（考虑添加毛玻璃效果组件、过渡动画和友好的 Loading 骨架屏）。
    > 3. 确保各个组件支持响应式设计。"

## 阶段二：Sprint 2 (AI 生成与测试循环)

这一阶段的核心是 AI Agent，我们需要确保 Prompt 都集中管理（遵守你 `Rule 9` 的规范）。

### 2.1 学习总结与闪卡 AI 生成器 (Study Summary & Flashcard Agent)

#### **步骤 1：后端 Agent 核心与 Context Builder (Backend)**
*   **任务**：实现合并材料与笔记的逻辑，并集成到生成器中。（注意：闪卡的骨架 API 已完成 `POST /modules/.../flashcards`）。
*   **AI 指令 (Prompt)**：
    > "我们正在建立后端 Agentic Content Generation 特性。
    > 1. 首先，请在 `server/utils/prompts.py` 中规范化注册我们所需的 '闪卡生成' 与 '知识总结' 的集中化 Prompt。
    > 2. 然后，用 TDD 方法实现合并对应材料与笔记文本的 Context Builder 逻辑。**非常重要：**必须在构建给大模型的 Context 时，强制注入该模块在阶段一设定好的 `'受众背景/情感敏感度'` 字段，作为系统级 Prompt 约束大模型的伦理与同理心边界。
    > 3. 完善并实现 '总结生成' 接口，确保请求外部 Claude 服务时使用 `async` 并配置了异常处理。"

#### **步骤 2：生成结果展示网页 (Frontend)**
*   **任务**：展示并渲染生成的闪卡与大纲。
*   **AI 指令 (Prompt)**：
    > "请基于后端的 `/flashcards` 接口，帮我实现一个精美的『闪卡学习』和『资料总结』双栏视图。
    > 要求具有极佳的视觉表现，比如闪卡的 3D 翻转动画、顺滑的卡片堆叠过渡效果。请一次性给我功能和高标准的 Tailwind UI。"

### 2.2 测验与反馈系统 (Quiz System & hints)

#### **步骤 1：测验生成后端与保存 (Backend)**
*   **任务**：生成多选题与开放题的题目及提示，并对这些生成结果实施 LLM-as-judge 评分。
*   **AI 指令 (Prompt)**：
    > "请通过 TDD 工作流，为我的 Quiz System 实现以下几个路由：
    > 1. 生成 Quiz 的接口（包含预定义的 hints 字段）。
    > 2. 保存学生测验提交结果的接口。
    > 3. 当任何 AI 测验被生成时，调用内置的 LLM-as-judge 函数记录质量评分入数据库。必须涵盖红绿重构过程。"

#### **步骤 2：测验互动 UI 与结果 (Frontend)**
*   **任务**：实现答题界面，评分反馈，带有 Eval 质量徽章的展示。
*   **AI 指令 (Prompt)**：
    > "帮我搭建测验的交互式 UI `QuizTakingView.tsx`。
    > 每次只出现一道题，并提供一个精美的“获取提示 (Hint)” 按钮。提交后，展示测验成绩以及大模型生成的个性化建议。界面需要生动有趣且反馈即时。"

## 阶段三：Sprint 3 (图表、运维与产品级封装)

最后阶段重在数据可视化与部署安全，这些通常在骨架建立后可以直接一起要求 UI。

### 3.1 教师报告与 MLOps 看板

*   **步骤 1：聚合分析 API (Backend)**
    > "请编写用于聚合分析的只读接口（包含平均分、知识盲点热力分布数据）。重点检查是否存在慢查询（N+1），通过 TDD 加入必要的压力或批量测试用例。"

*   **步骤 2：高级图表 UI (Frontend)**
    > "请引入类似于 Recharts 或 Chart.js 这类的库，为我们的 Instructor Report 和 MLOps 看板构建高级数据可视化。要求图表中除了包含班级评估成绩，还必须有大模型的质量趋势图表 (LLM Quality Trend Charts) 以及 LLM-as-judge 分析报告。考虑到暗色模式、数据进入时附带缓动加载动画（Fade in / Slide in）。"

### 3.2 生产环境准备 (Production Setup)
*   **系统检查与 CI/CD**

    > 2. 补全安全机制（如设置 Rate Limiter 防御），注入所有环境变量至 `.env`（切勿硬编码）。

*   **AI 指令 (Prompt)**：

    > 1. 一份详尽专业的 `README.md`，涵盖完整启动指南。
