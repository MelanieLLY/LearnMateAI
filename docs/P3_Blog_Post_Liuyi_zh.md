# LearnMate AI：借助 Claude Code 构建全栈教育平台的工程实践

在过去的开发周期中，我们的团队完成了 **LearnMate AI** 平台的构建。这是一个前后端分离的教育辅助平台（SaaS 应用）。本项目不仅实现了教师与学生端的核心业务逻辑，也是我们系统性地将 Claude Code（包括 Hooks、Custom Skills、MCP 和 Agents 等高级特性）与敏捷开发（Agile）、并行分支工作流（Git Worktree）深度集成的工程实践纪实。

本文将复盘我们在开发过程中，如何利用 AI 提升研发效率，同时通过建立防御性机制来解决底层技术陷阱与模型不稳定性问题。

---

## 🎯 架构重构与存储层优化

在项目初期（Sprint 1），我们优先进行了系统架构的审查与重构。我们发现了早期系统在前端路由与存储层存在的耦合问题。
借助 AI 的代码重构能力，我们以较低的成本完成了 Course 到 Module 的数据模型关联（级联更新），并实现了一个环境自适应的存储抽象层（Storage Abstraction）。该抽象层目前以本地静态目录作为默认存储，并为后期迁移至 AWS S3 预留了接口。同时，为了解决本地多服务启动时的冲突问题，我们编写了动态端口探测脚本，实现了 Vite 与 FastAPI 的跨进程端口自动发现与绑定。

---

## 🤖 工具链扩展：Claude Code 深度集成

本项目的一个核心技术点是将 `everything-claude-code` 插件生态整合进日常工作流，并解决了一系列底层配置问题：

### 1. 解决 Hook 的跨进程通信限制
为了在项目中强制执行 TDD（测试驱动开发）规范，我们尝试在 `.claude/settings.json` 中配置 `PreToolUse` Hook，以便在 Git Commit 前拦截未通过测试的代码。
在实际应用中，我们发现 Claude 对标准错误码（如 `exit 1`）的拦截处理存在局限，且进程间通信（IPC）有时会导致管道挂起。通过系统级排查，我们改进了拦截方案：采用 `read -t 1` 避免死锁，并在测试失败时强制返回 `exit 2` 并将信息重定向至 `stderr`。这一配置从进程级别有效阻断了未经测试的代码提交，保障了主干分支的代码质量。

### 2. 插件的本地化定制与自定义指令
为了实现对 `everything-claude-code` 核心组件的深度控制，我们跳过了默认的在线安装，直接提取核心规则并完成了高度定制化的本地部署。此外，针对后台脚本无法跨进程保存上下文对话历史的挑战，我们开发了专属的 `/chat-history-log` 自定义指令，以文件读写的方式规范化了对话日志的自动归档流程，完美解决了数据持久化的问题。

### 3. MCP 与 Agent 的自动化协作
在日常任务管理中，我们接入了 **GitHub MCP Server**，实现在终端内直接拉取和解析 Open Issues 状态。在文档维护环节，我们调用了本地配置的 `doc-reviewer` Agent，对 README 进行了自动化的格式审计与结构排查，大幅提升了文档的规范性。

## 🧪 保障数据一致性：处理模型输出的结构化错误

在集成大语言模型时，系统的健壮性面临了输入结构不可控的挑战。

在开发“测验生成”接口时，我们遇到了典型的数据不一致问题：前端成功展示了测验内容，但后端数据库并未落库，导致页面刷新后数据丢失。
经过全链路排查，我们定位到问题根源在于模型生成的 JSON 格式异常。当内容包含复杂的中文语境和标点时，模型偶尔会输出未经正确转义的字符串化 JSON（Stringified JSON），导致后端的 Pydantic 在反序列化时抛出 HTTP 500 异常。
为了修复此缺陷，我们优化了 `SYSTEM_PROMPT` 的结构约束，并在 Python 处理层引入了 `json-repair` 库进行容错解析。配合我们编写的 Playwright E2E（端到端）自动化测试，我们有效确保了测验和闪卡生成接口的高可用性。

## 🔀 并行开发与版本控制：Git Worktree 实践

在 Sprint 2 中，我们引入了 **Git Worktree**，使得 `feat/33-flashcard-ui` 与 `feat/24-quiz-ui` 实现了真正的并行开发。这两个独立的工作区分别承担了复杂的 3D 翻转动画（CSS + SVG 实现）以及测验系统状态机的构建。

在敏捷开发的高频合并过程中，多分支并行难免会遇到复杂的版本依赖与合并挑战（Merge Conflicts）。例如，在同步更新核心功能树时，远端与本地提交流产生了复杂的交叉。我们通过准确的拓扑结构分析，果断利用 `git reset --hard` 重置状态基线，并通过强力重写远端历史（Force Push）覆盖了混杂的提交记录，完美维护了 Git 历史的线性与整洁。

## 💡 结语

回顾 LearnMate AI 的开发过程，AI 工具极大地加速了代码编写，但也对我们的系统架构设计和故障排查能力提出了更高的要求。从处理跨进程拦截、大模型格式解析错误，到 Git 分支冲突与 E2E 测试环境配置，我们在利用 AI 赋能的同时，也建立起了更严谨的工程化防御思维。这套结合了自动化工具链与严格测试标准的工作流，将为我们未来的软件开发实践提供重要参考。
